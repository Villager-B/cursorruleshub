Cursorの.cursorrulesをGithubから検索して他のユーザの.cursorrulesを参考にすることができるサービスを作成してください．

# サービスの目的と対象ユーザーの明確化
- **目的**: ユーザーが他の開発者の`.cursorrules`を参考にすることで、効率的にCursorの設定をカスタマイズできるプラットフォームを提供する。
- **対象ユーザー**: Cursorを使用している開発者、特に独自のルールを設定したいが手間を省きたいと考えている人々。

# データの収集と管理
- GitHub APIを使用して公開リポジトリから`.cursorrules`ファイルを含むリポジトリ情報を取得する
  - スター数の多い上位100件のリポジトリのみを取得対象とする
- APIから取得したデータについて./data/に保存する
- 24hに一度，GitHub Actionsを使用して./data/を更新する
  - GitHub Actionsからpythonスクリプトを実行し，./data/を更新とGitHub Pagesへのデプロイを行う

#  UI/UXの設計
- 必要な機能
  - ソート機能
  - フィルタ機能
  - ページネーション機能
- ソートやフィルタができる対象は以下
  - スター数
  - リポジトリで使用されている主要言語
- 表示内容
    - リポジトリの概要，リポジトリのリンク，リポジトリのスター数，リポジトリで使用されている主要言語を表示する

# 技術制約
- ローカルでデータ収集プログラムの検証を実施したいのでvenvを使用してください．
- GitHub Pagesでのデプロイを行います．GitHub Pagesの制約に注意してください．
- デプロイにはGitHub Actionsを使用します．
- 必要なデータ収集プログラムについては，Pythonで作成してください．
